### 2. Excel 列头手动映射 UI (原则 #8)

> **设计参考**: 原则 #8, AGG-011 (FieldMapping)

当前所有导入均使用硬编码列名（如 `defect-governance.ts` 硬编码 `row.编号`, `row.等级`, `row.发现人`）。目标：
- 上传 Excel → 系统显示列头 → 管理员手动选择映射
- 映射配置保存（存入 ScoringRule.config）
- 同维度下次自动匹配（或管理员手动调整）

### 3. 11 个维度的自动导入 (PRD §7.7)

## ⚠️ 设计不一致需解决

### 预审行为的三种说法

| 来源 | 行为 | 影响 |
|------|------|------|
| PRD §3.5.4 | 预审不通过 = 拒绝提交（硬阻断） | 员工必须修正才能提交 |
| 设计文档 原则 #3 | 预审不通过 = 软提示（不阻断） | 人为审核兜底 |
| 代码实现 | 软提示 | 提交总是成功，附带 warnings |

**建议**: 统一设计文档和 PRD。如果选择"软提示"（推荐，因为入职日期数据可能不准确），需更新 PRD §3.5.4 的描述；如果选择"硬阻断"，需修改代码。
