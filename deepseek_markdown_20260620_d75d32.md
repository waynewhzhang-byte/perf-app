## 1. 项目背景与目标

### 1.1 背景
当前《基本素质信息》Excel工作簿中的“部门”（Sheet1）及“所属单位”（Sheet2）字段承载了多层级组织信息（包含工区、处室、班组），字段内涵混杂，不利于按特定工区维度进行筛选、统计与权限管控。

### 1.2 目标
- 新增标准字段 **“工区”** ，提取并规范核心生产工区名称。
- 对于包含“班组”信息的记录（Sheet2），额外提取标准 **“班组”** 字段。
- 保持原始数据完整性，仅新增列或对特定列进行标准化清空。

## 4. 详细转换逻辑规则

### 4.1 工作表1（技能等级 职称）转换规则

| 条件判断 | 工区值 | 部门值（原字段处理） |
| :--- | :--- | :--- |
| 部门字段 **等于** 白名单中的任一工区名称 | 赋值为该工区名称 | **清空**（置为Null/空字符串） |
| 部门字段 **不等于** 白名单（含空白、其他文字） | 赋值为 **“总部”** | **保留**原始内容不变 |

### 4.2 工作表2（考核结果）转换规则

该表处理“所属单位”字段。需区分**是否包含斜杠“/”**（用于分割工区/处室与班组）：

#### 情况A：字段中包含“/”（如 `晋北运维分部/变电运维一班`）

| 斜杠左侧（Left部分）判断 | 工区值 | 部门值（新增列） | 班组值（新增列） |
| :--- | :--- | :--- | :--- |
| Left部分 **等于** 白名单 | 赋值为该工区名称 | **置空** | 赋值为斜杠右侧（Right部分） |
| Left部分 **不等于** 白名单 | 赋值为 **“总部”** | 赋值为Left部分 | 赋值为斜杠右侧（Right部分） |

#### 情况B：字段中**不包含**“/”

| 整体值判断 | 工区值 | 部门值（新增列） | 班组值（新增列） |
| :--- | :--- | :--- | :--- |
| 整体值 **等于** 白名单 | 赋值为该工区名称 | **置空** | **置空** |
| 整体值 **不等于** 白名单 | 赋值为 **“总部”** | 赋值为整体原始值 | **置空** |

> **重要约束**：原“所属单位”列在Sheet2中**永久保留**，不做任何删除或清空操作。

## 6. 异常与空值处理规范

| 场景 | 处理策略 |
| :--- | :--- |
| 源字段为 `Null` 或空字符串 | 工区赋值为“总部”，部门/班组均置为空字符串。 |
| 斜杠“/”左侧或右侧存在首尾空格 | 强制执行 `.strip()` 去除空格后再匹配与赋值。 |
| 字段包含多个“/”（如 `A/B/C`） | 仅以**第一个“/”**为分割点，左侧部分作为工区/部门依据，右侧剩余全部（含斜杠）作为班组值。 |
| 匹配白名单时大小写差异 | 白名单为纯中文，不存在大小写问题；若含有英文，建议统一转为全角或半角后比对（本方案中不涉及）。 |

### 7.2 输出检查点（质量门禁）
1. **Sheet1**：所有属于12个工区的记录，“部门”列必须为空，且“工区”列有值。
2. **Sheet2**：所有 `所属单位` 含“/”且左部为工区名的记录，“班组”列必须成功提取右侧值。
3. **完整性**：两个Sheet的记录总数必须与原文件完全一致（`count`相等）。

WHITELIST = [
    "变电检修中心", "二次检修中心", "晋北运维分部", "晋南运维分部", "晋中运维分部",

]

    if dept in WHITELIST:
        return dept, ""  # 工区, 新部门(清空)
    else:
        return "总部", dept  # 工区, 新部门(保留)

        else:
            return "总部", left, right
    else:
